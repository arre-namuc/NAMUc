# CutFlow 배포 가이드

## 전체 순서 요약
1. GitHub 저장소 생성 → 파일 업로드
2. Firebase 프로젝트 생성
3. Vercel 배포 + 환경변수 입력

예상 소요 시간: **약 30~40분**

---

## STEP 1 — GitHub 저장소 만들기

1. https://github.com 접속 → 로그인
2. 우상단 `+` 버튼 → `New repository`
3. Repository name: `cutflow`
4. `Public` 선택
5. `Create repository` 클릭
6. 저장소 페이지에서 `uploading an existing file` 클릭
7. **이 폴더 안의 파일 전체를 드래그 앤 드롭** (숨김파일 포함: `.gitignore`, `.env.example`)
8. `Commit changes` 클릭

> ⚠️ `.env` 파일은 절대 올리지 마세요 (`.env.example` 은 괜찮습니다)

---

## STEP 2 — Firebase 설정

### 2-1. 프로젝트 생성
1. https://console.firebase.google.com 접속 (구글 계정으로 로그인)
2. `프로젝트 추가` → 이름: `cutflow` → 계속 → 완료

### 2-2. Firestore 데이터베이스 생성
1. 왼쪽 메뉴 `빌드` → `Firestore Database`
2. `데이터베이스 만들기`
3. **테스트 모드에서 시작** 선택 (나중에 보안 규칙 강화 가능)
4. 위치: `asia-northeast3 (서울)` 선택 → 완료

### 2-3. Storage 생성 (파일 첨부용)
1. 왼쪽 메뉴 `빌드` → `Storage`
2. `시작하기` → 테스트 모드 → 위치 `asia-northeast3` → 완료

### 2-4. 웹 앱 등록 및 설정값 복사
1. 프로젝트 홈 → `</>` (웹) 아이콘 클릭
2. 앱 닉네임: `cutflow-web` → `앱 등록`
3. 아래 코드에서 값들을 복사해 둡니다:
```
apiKey: "AIza..."
authDomain: "cutflow-xxx.firebaseapp.com"
projectId: "cutflow-xxx"
storageBucket: "cutflow-xxx.appspot.com"
messagingSenderId: "123456789"
appId: "1:123456789:web:abc..."
```

---

## STEP 3 — Vercel 배포

1. https://vercel.com 접속 → `Sign Up with GitHub`
2. `New Project` → GitHub에서 `cutflow` 저장소 선택 → `Import`
3. **Environment Variables** 섹션에서 아래 값들을 하나씩 추가:

| Name | Value |
|------|-------|
| `VITE_ANTHROPIC_KEY` | `sk-ant-api03-...` |
| `VITE_FIREBASE_API_KEY` | Firebase에서 복사한 apiKey |
| `VITE_FIREBASE_AUTH_DOMAIN` | Firebase authDomain |
| `VITE_FIREBASE_PROJECT_ID` | Firebase projectId |
| `VITE_FIREBASE_STORAGE_BUCKET` | Firebase storageBucket |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | Firebase messagingSenderId |
| `VITE_FIREBASE_APP_ID` | Firebase appId |

4. `Deploy` 클릭
5. 2~3분 후 `https://cutflow-xxx.vercel.app` 주소 생성!

---

## STEP 4 — 팀원 공유

생성된 Vercel 주소를 팀원들에게 공유하세요.
앱 내 계정/비밀번호는 `src/App.jsx` 상단 `ACCOUNTS` 배열에서 수정할 수 있습니다.

---

## 이후 업데이트 방법

기능 개선이 필요할 때:
1. Claude에게 수정 요청 → 새 `App.jsx` 파일 받기
2. GitHub 저장소 → `src/App.jsx` 클릭 → 연필 아이콘(Edit)
3. 새 내용으로 교체 → `Commit changes`
4. Vercel이 자동으로 재배포 (약 1~2분)

---

## 문제 해결

**배포 후 화면이 안 나올 때**
→ Vercel 대시보드 → 프로젝트 → `Deployments` → 빨간 빌드 클릭 → 오류 메시지 확인

**데이터가 저장 안 될 때**
→ 헤더에 `⚠️ 로컬 모드` 표시되면 Firebase 환경변수 미입력 상태
→ Vercel → Settings → Environment Variables 재확인 후 Redeploy

**AI 분석이 안 될 때**
→ `VITE_ANTHROPIC_KEY` 환경변수 확인
→ https://console.anthropic.com 에서 키 유효 여부 확인
